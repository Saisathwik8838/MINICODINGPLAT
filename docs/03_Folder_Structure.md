# 3. Folder Structure

## Architectural Rationale for this Structure:

1.  **Separation of Concerns**: Each major component (Frontend, Backend API, asynchronous Worker, isolated Executor environments, and Infrastructure configuration) lives in its own dedicated top-level directory. This clearly demarcates responsibilities.
2.  **Scalability**: By physically separating the API (`backend/`) from the execution engine (`worker/` & `executor/`), we can deploy, scale, and manage these services independently depending on the load.
3.  **Security boundaries**: The `executor/` folder uniquely houses the highly scrutinized Dockerfiles and runner scripts that establish the restricted environment. This separation forces a clear boundary between standard business logic and sensitive execution logic.
4.  **DevOps Integration**: The `infrastructure/` directory centralizes all Docker Compose, Nginx proxy, and monitoring configurations, providing a clean path for CI/CD deployment (`Deploy.yml`) to orchestrate the entire platform uniformly.
